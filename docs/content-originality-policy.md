# Content Originality Policy

This repository must be useful, public-safe, and contribution-friendly.

## What We Do

- Write original learner-friendly explanations.
- Use official documentation as the primary reference.
- Link to useful external repositories and articles with credit.
- Summarize concepts in our own structure and examples.
- Create our own labs, interview questions, projects, and troubleshooting notes.
- Display external images only when credited and used as references.

## What We Do Not Do

- Do not copy an entire third-party repository.
- Do not rephrase a whole external repo page-by-page.
- Do not upload pirated books, paid course material, exam dumps, or leaked content.
- Do not copy diagrams unless the license allows it and credit is provided.
- Do not remove copyright notices from referenced material.

## Safe Contribution Rule

If you learn from a source, write your own explanation from understanding:

```text
Read -> Practice -> Close source -> Write from memory -> Add credit link -> Validate URL
```

## Reference Sources

Use sources in this order:

1. Official documentation
2. CNCF, OWASP, Linux Foundation, and cloud provider material
3. Open source project docs
4. Trusted engineering blogs
5. External GitHub repositories as inspiration and linked references, not copied content

## Why This Matters

Learners should build real understanding, and contributors should be comfortable sharing this repo publicly without copyright or licensing problems.
