# Contribution Workflow

This repo should grow through small, clear, validated contributions.

## Contribution Types

- Notes
- Labs
- Commands
- Interview questions
- Diagrams
- Project ideas
- Troubleshooting logs
- URL updates

## Workflow

1. Pick one topic folder.
2. Read the existing README.
3. Add a focused note or lab.
4. Validate URLs.
5. Add cleanup steps for cloud resources.
6. Open a pull request using the template.

## Good Contribution Example

```text
Add Kubernetes readiness and liveness probe lab with commands, YAML, expected output, failure case, and cleanup command.
```

## Bad Contribution Example

```text
Add 100 random Kubernetes links with no explanation.
```

## Review Checklist

- Is it accurate?
- Is it learner-friendly?
- Are commands tested or clearly marked as examples?
- Are URLs validated?
- Is copyrighted material avoided?
- Are cleanup steps included?
