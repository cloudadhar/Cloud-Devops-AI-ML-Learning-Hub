# How to Use This Repository

This repository is designed for learners who want a clear path from basics to job-ready projects across Cloud, DevOps, DevSecOps, AI/ML, MLOps, Linux, and full-stack engineering.

## For Beginners

1. Open [STUDENT_START_HERE.md](../STUDENT_START_HERE.md).
2. Study [Foundation Knowledge Map](foundation-knowledge-map.md).
3. Learn Linux, networking, Git, and basic programming before jumping into cloud tools.
4. Build one small project every 2 weeks.
5. Keep notes using [Learning Log Template](../logs/learning-log-template.md).

## For Intermediate Learners

1. Pick a target role: cloud engineer, DevOps engineer, DevSecOps engineer, MLOps engineer, or full-stack developer.
2. Follow the relevant roadmap in [ROADMAP_INDEX.md](../ROADMAP_INDEX.md).
3. Study one tool stack from [stacks/](../stacks/README.md).
4. Build one project that touches CI/CD, Docker, cloud, logs, monitoring, and security.
5. Prepare interview answers from [Interview Preparation Guide](interview-preparation-guide.md).

## For Advanced Learners

1. Choose production-style projects with real tradeoffs.
2. Add architecture diagrams, Terraform modules, secrets handling, monitoring, rollback, and cleanup.
3. Write incident reports and troubleshooting logs.
4. Contribute advanced notes back to topic folders.

## Recommended Daily Routine

- 30 minutes reading official docs.
- 60 minutes hands-on practice.
- 20 minutes documenting notes and errors.
- 10 minutes reviewing interview questions.

## What Not to Do

- Do not learn random tools without understanding the lifecycle.
- Do not copy commands without understanding what they change.
- Do not create cloud resources without cleanup steps.
- Do not add links without checking them.
- Do not store secrets in Git.
