# Portfolio and GitHub Profile Guide

Your portfolio should prove that you can build, document, troubleshoot, and explain systems.

## GitHub Profile Must Have

- Clear profile README
- Pinned projects
- Good commit history
- Project READMEs
- Screenshots or diagrams
- Setup and cleanup steps
- What you learned

## Project README Structure

```md
# Project Name

## Problem

## Architecture

## Tools Used

## How to Run

## CI/CD

## Security

## Monitoring

## Cleanup

## Troubleshooting

## What I Learned
```

## Projects to Build

### Beginner

- Static website deployed on cloud.
- Linux command cheat sheet.
- GitHub Actions hello pipeline.

### Intermediate

- Dockerized API behind Nginx.
- Terraform cloud VM deployment.
- CI/CD pipeline with tests and scan.
- Prometheus and Grafana dashboard.

### Advanced

- Kubernetes app with Helm and Argo CD.
- API gateway using Kong.
- DevSecOps pipeline with SonarQube, Trivy, and Checkov.
- RAG chatbot with vector database and monitoring.
- MLOps model API with MLflow and deployment.

## Resume Bullet Pattern

Use this pattern:

```text
Built <system> using <tools> to achieve <outcome>, including <security/monitoring/automation proof>.
```

Example:

```text
Built a Dockerized FastAPI service deployed through GitHub Actions and Terraform on AWS EC2, using Nginx reverse proxy, Trivy image scanning, and CloudWatch logs for operational visibility.
```
