# URL Research and Validation Policy

This repository should stay useful and trustworthy. Every major link should be checked before it is added.

## Preferred Sources

1. Official product documentation.
2. Official open source project documentation.
3. CNCF, OWASP, Linux Foundation, cloud provider docs.
4. Vendor engineering blogs when they explain a practical tool or pattern.
5. Community blogs only when they add a clear lab or troubleshooting value.

## Avoid

- Pirated PDFs.
- Exam dumps.
- Copied paid course material.
- Outdated tutorials with broken commands.
- Blog posts that only repeat marketing content.

## Validation Checklist

- URL opens successfully.
- Source is official or trusted.
- Date checked is recorded.
- Link text clearly describes the page.
- Notes explain why the link helps learners.

## Where to Record Links

- Use [URL Validation Log](../logs/url-validation-log.md) for major repo-wide links.
- Use the relevant topic README for topic-specific links.
- Use [Research Notes](../logs/research-notes.md) for source decisions.
