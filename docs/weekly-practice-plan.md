# Weekly Practice Plan

Use this repeatable weekly rhythm for any topic.

## Monday: Concept

- Read one official documentation page.
- Write a 10-line summary.
- List unknown words.

## Tuesday: Commands

- Practice commands.
- Save useful commands in the topic folder.
- Record errors.

## Wednesday: Mini Lab

- Build a small example.
- Keep it simple and working.
- Commit the code.

## Thursday: Debugging

- Break the lab intentionally.
- Use logs and docs to fix it.
- Write a troubleshooting note.

## Friday: Security and Cost

- Ask: What can go wrong?
- Check permissions, secrets, exposed ports, and cloud cost.
- Add cleanup steps.

## Saturday: Interview

- Write 5 interview questions.
- Answer them in your own words.
- Record a short explanation if possible.

## Sunday: Portfolio

- Update README.
- Add architecture diagram.
- Push to GitHub.
- Plan next week.
