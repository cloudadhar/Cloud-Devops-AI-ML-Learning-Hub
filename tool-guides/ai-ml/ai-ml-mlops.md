# AI, ML, and MLOps Guide Index

These are related topics, but learners should not study them as one mixed subject. Use the separate files below.

| Topic | Best For | Guide |
| --- | --- | --- |
| Artificial Intelligence Engineering | Building AI applications, APIs, assistants, and prompt/model workflows | [Artificial Intelligence Engineering](artificial-intelligence.md) |
| Machine Learning | Training and evaluating predictive models | [Machine Learning](machine-learning.md) |
| MLOps | Deploying, monitoring, governing, and retraining models in production | [MLOps](mlops.md) |
| RAG and Agentic AI | Document Q&A, vector search, tool calling, memory, and agent workflows | [RAG and Agentic AI](rag-agentic-ai.md) |
| Python and FastAPI | API foundation for AI/ML projects | [Python and FastAPI](python-fastapi.md) |

## How To Choose

- If you want to build AI apps quickly, start with AI Engineering.
- If you want to understand model training, start with Machine Learning.
- If you want to deploy models safely, start with MLOps.
- If you want to build document Q&A or agents, start with RAG and Agentic AI.

## Shared Production Flow

```text
Data/knowledge -> model or retrieval system -> API -> gateway -> monitoring -> evaluation -> improvement loop
```

## Interview Direction

Do not answer only with definitions. Explain architecture, data, failure mode, monitoring, security, and rollback.
