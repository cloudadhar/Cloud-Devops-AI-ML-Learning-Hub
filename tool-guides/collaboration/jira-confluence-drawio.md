# Jira, Confluence, and draw.io Zero to Hero Guide

## What It Is

Jira tracks work, Confluence documents knowledge, and draw.io creates architecture diagrams. Together they help teams plan, communicate, and preserve decisions.

## Why It Matters

- Helps learners understand where the tool fits in real engineering work.
- Connects theory with projects, interviews, and production troubleshooting.
- Gives a path from beginner usage to advanced integration.

## Short History

Atlassian Jira and Confluence became common tools for agile software teams and enterprise documentation. draw.io, also known as diagrams.net, became popular for lightweight architecture and workflow diagrams.

## Architecture and Core Concepts

Collaboration architecture:

```text
Idea -> Jira epic/story/task -> Confluence design doc -> draw.io architecture -> PR -> Release notes
```

Key concepts:

- Jira: project, epic, story, task, bug, sprint, board, workflow.
- Confluence: page, space, template, decision record, runbook.
- draw.io: architecture diagram, flowchart, sequence, network diagram.

## Zero to Hero Learning Path

1. Create Jira issue types and board.
2. Write acceptance criteria.
3. Create Confluence design document.
4. Draw architecture in draw.io.
5. Link docs to Jira tickets.
6. Use release notes and runbooks.
7. Create incident postmortem pages.
8. Use decision records for tool choices.

## How to Start Using It

Useful workflow:

```text
Jira ticket -> Design doc -> Diagram -> Implementation PR -> Deployment -> Runbook update
```


## Common Integrations and Pipeline Usage

Integrates with GitHub, GitLab, Slack, Microsoft Teams, Confluence macros, incident tools, CI/CD notifications, and architecture documentation.

## Advanced Topics

- Agile workflows and custom statuses.
- Architecture decision records.
- Runbooks and incident postmortems.
- Requirements traceability.
- Release management.
- Diagram versioning.

## Hands-on Project

Document a project from idea to deployment: Jira ticket, Confluence architecture page, draw.io diagram, GitHub PR, and release note.
